# Second Post

This is our second blog post, covering some advanced features and tips for using the documentation site.

## Advanced Tips

### Keyboard Shortcuts

- `Ctrl+K` / `Cmd+K` - Focus search
- `Esc` - Close search results
- Click hamburger menu to toggle sidebar

### Content Organization

All content is organized in the `content/` directory:

```
content/
├── _config.md      # Site configuration
├── _header.md      # Header content
├── _footer.md      # Footer content
├── logo.svg        # Site logo
├── home/           # Homepage content
├── about/          # About pages
├── blog/           # Blog posts
└── docs/           # Documentation
```

### Adding New Content

1. Create a new markdown file in the appropriate directory
2. Update the `CONTENT_STRUCTURE` in `index.html`
3. Your content will automatically appear in the sidebar

## Best Practices

- Use clear, descriptive headings
- Keep paragraphs concise
- Add code examples where helpful
- Include links to related content
